# П’ятнашки (Puzzle Slider)

Гра **П’ятнашки** — це головоломка у вигляді поля 4×4 з 15 плитками та однією порожньою клітинкою.  
Мета гри — зібрати плитки в правильному порядку.

# Лабораторна робота №1

Було зібрано каркас гри **П’ятнашки**:

- створено базові сторінки: `StartPage`, `GamePage`, `ResultsPage`
- додано базові стани-плейсхолдери для кожної сторінки
- додано початкові стилі для гри
- створено компоненти для перевикористання: `Layout`, `Button`, `Tile`, `Header`, `Board`
- забезпечено чіткий компонентно-орієнтований підхід та логічну структуру проєкту


## Створені сторінки

### StartPage
Сторінка старту гри з кнопкою для початку та індикатором стану гри.

### GamePage
Сторінка гри з ігровим полем, таймером, інформацією про поточний стан гри та кількістю правильних ходів.

### ResultsPage
Сторінка з результатами користувача після завершення гри.

# Лабораторна робота №2

## Додана логіка гри за допомогою хуків
На наступному етапі реалізовано **основну логіку програми**, при цьому компоненти залишаються максимально чистими.  
Уся бізнес-логіка винесена до **кастомних хуків**, що забезпечує розділення відповідальності.
### useGameLogic
Хук відповідає за:
- ініціалізацію ігрового поля (масив із 15 чисел і порожньої клітинки);
- логіку руху плиток при натисканні;
- підрахунок кількості ходів;
- перевірку завершення гри;
- зміну стану гри (`idle`, `playing`, `finished`).

> Цей хук реалізує повну логіку ігрового процесу, а компонент `GamePage` лише відображає дані та реагує на дії користувача.

### useTimer
Хук реалізує таймер, який:
- запускається на початку гри;
- зупиняється після завершення;
- може бути скинутий для нової гри.

> Такий підхід дозволяє незалежно керувати логікою часу без втручання у компонент сторінки.

## Покращення логіки завершення гри та відображення результатів

На останньому етапі додано можливість:

- Натискання кнопки **"Завершити"** до повного збору плиток:
  - Гра зупиняється;
  - Відображається час гри та кількість ходів;
  - Показується повідомлення, що плитки не зібрані правильно.
- Автоматичне визначення виграшу:
  - Якщо всі плитки зібрані правильно, гра завершується;
  - На сторінці результатів відображається час, кількість ходів та повідомлення-вітання.
- Виведення кількості зроблених ходів на сторінці гри.

### Оновлення компонентів

- **GamePage**:
  - Повністю чистий UI-компонент;
  - Використовує хуки `useGameLogic` та `useTimer` для керування станом гри та часу;
  - Передає дані до сторінки результатів через колбек `onFinish({success, time, moves})`.

- **ResultsPage**:
  - Отримує об'єкт `results` і відображає:
    - Час гри;
    - Кількість ходів;
    - Повідомлення про результат (успіх/неуспіх).

### Принцип роботи

- Вся бізнес-логіка зосереджена в хуках:
  - `useGameLogic` — керує плитками, ходами та статусом гри;
  - `useTimer` — керує відліком часу.
- Компоненти та сторінки залишаються максимально легкими і відповідають лише за рендеринг та події користувача.
- Забезпечено **компонентно-орієнтований підхід** та перевикористання компонентів.

# Лабораторна робота №3

## Додано сторінку SettingsPage із використанням бібліотек:

- `Formik` - для створення форми
- `Yup` - для валідації введених даних

На сторінці можна:
- Обрати рівень складності (easy, normal, hard)

- Увімкнути або вимкнути режим для новачка

- Налаштування зберігаються у localStorage через контекст SettingsContext

Контекст SettingsContext дозволяє отримати доступ до налаштувань із будь-якої частини застосунку.

## Режим для новачка (`beginnerMode`)

У цьому режимі поле генерується як вирішуване, але просте:
гру починають з майже зібраного стану, створеного через кілька випадкових простих ходів.
Такий підхід гарантує:

- вирішуваність комбінації

- мінімальну складність

- плавне входження у геймплей

## Модальне вікно завершення гри

Додано компонент `GameOverModal` з використанням React-порталів.
Після завершення гри або натискання “Завершити” відображається модальне вікно, де показано:

- результат (успіх чи ні)

- кількість ходів

- час проходження

- кнопки "Почати заново" та "Вийти"

## Збереження гри

- Усі налаштування гри (difficulty, beginnerMode) зберігаються в `localStorage`

- Контекст автоматично зчитує їх при перезапуску застосунку

# Лабораторна робота №4

## Додано роутинг між сторінками

Реалізовано навігацію між сторінками за допомогою React Router v6:

- / — стартова сторінка з полем введення імені користувача;

- /settings — сторінка налаштувань гри;

- /game/:username — динамічна сторінка гри для конкретного користувача.

Роутинг динамічний — ім’я користувача передається як параметр (:username), зберігається у localStorage та відображається у грі.

## Стилізація застосунку за допомогою `styled-components`

Усі компоненти переписано з використанням styled-components — кожен має власний файл стилів із темною мінімалістичною палітрою.

Забезпечено повну стилізацію:

- адаптивне розміщення плиток у сітці;

- приємна типографіка;

- кнопки з hover-ефектами;

- модальне вікно з плавною появою.

# Лабораторна робота №5

## Перехід на Zustand

Замість контексту (`SettingsContext`) використано Zustand для глобального керування станом:
- settings — рівень складності, режим новачка;
- results — масив результатів користувачів;

Функції:
- setSettings(newSettings) — змінює налаштування гри та зберігає в localStorage;

- addResult(result) — додає новий результат гри;

- автоматичне збереження результатів у localStorage.

Основна логіка цього стейт-менеджеру знаходиться у файлі gameStore.js

## Створено таблицю результатів (`ResultsPage`)

Створено сторінку результатів гри з відображенням усіх сесій користувачів.

Показується:
- Гравець	(Ім’я користувача)
- Розмір поля	(3×3, 4×4 або 5×5)
- Режим новачка	(Так / Ні)
- Час	(Час проходження гри)
- Ходи	(Кількість зроблених ходів)
- Дата	(Дата гри)
- Успіх	(Так / Ні)
